Modifications Report
PalmMapBot backend comparison deliverable - generated 9 May 2026
Comparison basis: the uploaded modified archive was compared against its Git baseline/HEAD for the linked PalmMapBot repository. The working tree contains local backend modifications on top of commit 5b12bd5. Python syntax compilation was also checked for the changed Python files and passed.

1. Executive summary

The original dashboard was mainly a Flask database/map page with mission buttons. The modified version turns it into a live backend dashboard with sensors, camera streaming, AI inference status, and real detection logging.
Fake/simulated tree detections were removed from the mission path. Tree detections are now recorded only when the camera/YOLO pipeline detects a palm tree during an active mission and a GPS fix is available.
The Raspberry Pi hardware layer was added through a new sensors package: USB/V4L2 camera, GPS serial reader, and TF-Luna LiDAR I2C reader.
A new NCNN-format palm tree detector model was added, and the dashboard now points to models/palm_tree_detector_ncnn_model instead of the original .pt-only flow.
The requirements were adjusted by commenting out PyQt5 packages and adding separate lightweight/essential requirement files for Raspberry Pi deployment.

3. Mission controller changes

Original behavior: pressing Start Mission triggered hard-coded survey waypoints, automatically simulated a tree detection at every waypoint, returned home, and completed the mission. This made the database fill with fake detections even when no real palm was detected.
Modified behavior: survey_farm() now only follows waypoint positions. It no longer calls tree_mapper.process_tree_detection() by itself. A new record_tree_detection() method was added for real detections only.

Changed logic | Original | Modified
survey_farm() | Followed waypoints and created simulated detections with confidence 0.90. | Follows waypoints only; does not create fake detections.
Detection logging | Tied to waypoint simulation. | Moved to record_tree_detection(), which requires an active mission.
Database effect | Every survey waypoint could create/merge a tree record. | Tree records are created only when the detector confirms a palm and GPS is valid.

New backend method added:
record_tree_detection(robot_x, robot_y, robot_yaw_rad, gps_lat, gps_lon, confidence)

Purpose:
- checks that a mission is active
- calls tree_mapper.process_tree_detection()
- returns the mapping result
- prints "Mapped real tree" instead of "Mapped tree"

4. Dashboard/backend integration changes

DB_PATH changed from the relative string data/palms.db to os.path.join(BASE_DIR, "data", "palms.db"), making database access more stable when running the app from different working directories.
Sensor imports are wrapped in try/except blocks. If LiDAR, GPS, or camera imports fail, the dashboard still starts and returns fallback JSON with an error message instead of crashing.
Ultralytics import is optional. If YOLO cannot be imported, the app reports AI unavailable instead of stopping the backend.
The model path now points to models/palm_tree_detector_ncnn_model.
The dashboard maintains latest_ai state containing model load status, palm detection flag, confidence, logging state, GPS state, last analysis time, last logged detection time, and model errors.
A background AI thread repeatedly grabs the latest camera frame, runs inference, annotates frames, updates the dashboard, and logs detections when allowed.
An MJPEG endpoint /video_feed streams the latest annotated frame to the browser.
The Start Mission button now only starts a mission; it no longer automatically surveys, returns home, and completes. A separate Complete Mission button was added.

6. AI detection and logging flow

Camera frame from sensors/camera_reader.py
    -> dashboard/app.py analyze_latest_frame()
        -> load_yolo_model() using models/palm_tree_detector_ncnn_model
        -> model(frame, conf=PALM_CONF_TH)
        -> update latest_ai status and annotated MJPEG frame
        -> if palm detected: maybe_log_detection(confidence)
            -> require cooldown to pass
            -> require active mission
            -> require GPS fix + latitude + longitude
            -> controller.record_tree_detection(...)
                -> tree_mapper.process_tree_detection(...)
                -> database stores/updates tree + detection

Runtime variable | Default | Role
PALM_CONF_TH | 0.25 | Minimum model confidence for detection.
PALM_LOG_COOLDOWN | 4.0 seconds | Minimum time between database logs to avoid duplicate detections.
AI_INTERVAL | 0.1 seconds | Delay between AI inference cycles.
MJPEG_FPS | 8.0 FPS | Frame rate for browser MJPEG stream.

11. Practical conclusion

The backend now works as a real hardware-aware Flask backend instead of a simulation-only dashboard. The most important functional change is the separation between mission movement and tree detection logging. A mission can be started and kept active, while the camera/AI/GPS pipeline records tree detections only when real conditions are satisfied.
