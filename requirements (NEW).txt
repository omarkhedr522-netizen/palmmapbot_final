2. Final recommended runtime requirements

For the Raspberry Pi backend/dashboard, do not blindly install the full exported requirements.txt. It is a large UTF-16 environment snapshot and includes training/desktop packages that are not needed for the live backend. Use a small runtime requirements file instead.
# requirements-raspi-runtime.txt
Flask
numpy
opencv-python-headless
pyserial
smbus2
ultralytics
ncnn
requests
pandas
Notes about this list:
Flask is required because dashboard/app.py is a Flask web server.
numpy and OpenCV are required for camera frames, JPEG encoding, placeholder images, and AI frame processing.
opencv-python-headless is preferred on Raspberry Pi for a web dashboard because the code does not use cv2.imshow(). Use opencv-python only if a desktop GUI/OpenCV display window is needed.
pyserial is required by sensors/gps_reader.py.
smbus2 is required by sensors/lidar_reader.py.
ultralytics is required by dashboard/app.py to load YOLO.
ncnn should be installed because the modified model is stored as an NCNN export and the included model_ncnn.py imports ncnn. If Ultralytics loads the NCNN folder without a separate ncnn install, this can be left installed harmlessly.
requests and pandas are not core to the live camera/GPS/LiDAR loop, but they are safe to keep because existing project scripts commonly use them.

Run these from the project folder on the Raspberry Pi. Adjust the virtual environment name if you prefer a different one.

5. System-level packages and Raspberry Pi interfaces

sudo apt update
sudo apt install -y python3-venv python3-pip i2c-tools libgl1 libglib2.0-0

# Enable I2C for the TF-Luna LiDAR
sudo raspi-config
# Interface Options -> I2C -> Enable

# Check I2C devices
sudo i2cdetect -y 1

# Check camera device
ls /dev/video*

# Check GPS serial device
ls /dev/ttyACM* /dev/ttyUSB* 2>/dev/null
LiDAR code assumes I2C bus 1 and TF-Luna address 0x10. If i2cdetect shows a different address or no device, fix wiring/interface settings before running the dashboard.
Camera code assumes the camera appears as /dev/video0. If the external USB camera appears as /dev/video1, update sensors/camera_reader.py or add a configurable environment variable later.
GPS code assumes /dev/ttyACM0 at 9600 baud. If the GPS appears as /dev/ttyUSB0, update GPSReader(port="/dev/ttyUSB0").
The Flask server binds to 0.0.0.0:5000, so another computer on the same network can open http://<raspberry-pi-ip>:5000.

6. Runtime environment variables

Variable | Default | When to change it
PALM_CONF_TH | 0.25 | Increase it if false detections happen; decrease it if the model misses palms.
PALM_LOG_COOLDOWN | 4.0 | Increase it if duplicate database logs happen while the camera sees the same tree.
AI_INTERVAL | 0.1 | Increase it on weak hardware to reduce CPU load; decrease it for faster inference checks.
MJPEG_FPS | 8.0 | Increase for smoother browser stream; decrease if browser/server becomes laggy.

# Example: run with safer logging and smoother stream
export PALM_CONF_TH=0.35
export PALM_LOG_COOLDOWN=6
export AI_INTERVAL=0.15
export MJPEG_FPS=10
python dashboard/app.py

7. Model files that must stay in place

Keep the folder name exactly as models/palm_tree_detector_ncnn_model because dashboard/app.py points to that path.
Required NCNN model assets: model.ncnn.param, model.ncnn.bin, and metadata.yaml.
The metadata identifies one class: palm_tree.
Do not rename the model folder unless MODEL_PATH in dashboard/app.py is updated too.

8. Important installation considerations

The current requirements.txt and requirements-lite.txt are UTF-16 encoded. Some pip workflows expect UTF-8, so create a fresh UTF-8 requirements-raspi-runtime.txt for deployment.
Avoid copying virtual environment folders between machines. Recreate the venv on the Raspberry Pi and reinstall packages there.
Do not commit palm_env/, vision313/, __pycache__/, *.pyc, or data/palms.db.
If OpenCV pip installation fails on the Raspberry Pi, try installing system OpenCV instead, then remove opencv-python-headless from the pip runtime file.
If Ultralytics crashes with illegal instruction or backend errors, keep the NCNN model path and try the smallest possible runtime dependency set. Avoid full torch/torchvision unless absolutely required on that device.
Run python -m py_compile on changed files after edits to catch syntax mistakes before launching the dashboard.
